- **覆盖**：完整用户流程

## 风险和注意事项

### 高风险项

1. **E2E 测试并行执行冲突**
   - 解决方案：使用测试 ID 前缀或独立事务
   - 备选方案：串行执行（已配置 `fullyParallel: false`）

2. **数据库迁移**
   - 确保测试数据库 Schema 与生产一致
   - 使用 Drizzle 迁移工具

3. **外部服务依赖**
   - Mock 短信、邮件、Turnstile、OAuth
   - 使用测试账号或 Mock 服务

### 中风险项

1. **携程功能对比**
   - 需要实际访问携程网站对比
   - 可能需要调整设计

2. **性能问题**
   - 测试数据库可能较慢
   - 需要优化查询和索引
