# 设计系统

欢迎来到 Nomad 项目的设计系统文档。

## 概述

本设计系统提供了一套完整的设计规范和组件库，确保整个应用程序的视觉一致性和用户体验。

## 设计原则

### 1. 一致性
在整个应用程序中保持视觉和交互的一致性，让用户能够快速学习和使用。

### 2. 无障碍性
确保所有用户，包括使用辅助技术的用户，都能轻松访问和使用我们的产品。

### 3. 响应式
设计适应各种屏幕尺寸和设备类型，提供优质的移动和桌面体验。

### 4. 简洁性
保持界面简洁清晰，避免不必要的复杂性，让用户专注于核心任务。

### 5. 性能优先
确保快速加载和流畅的交互体验。

## 设计系统组成

### 🎨 基础
- **颜色** - 颜色系统和使用指南
- **字体** - 字体选择和排版规范
- **间距** - 布局和间距系统
- **圆角** - 边框圆角规范

### 🧩 组件
- **通用组件** - 按钮、输入框、卡片等基础组件
- **表单组件** - 表单相关的专用组件
- **航班组件** - 航班搜索、预订相关组件
- **认证组件** - 登录、注册相关组件

### 📐 模式
- **布局模式** - 常见的页面布局方案
- **交互模式** - 常见的交互行为规范
- **导航模式** - 导航和路由规范

## 技术栈

本设计系统基于以下技术构建：

- **React 19** - UI 框架
- **Next.js 15** - 应用框架
- **Tailwind CSS 4** - 样式系统
- **Radix UI** - 无障碍组件基础
- **OKLCH** - 颜色空间

## 如何使用

### 开发者

查看各个组件的文档和示例代码，直接在项目中引入使用：

```tsx
import { Button } from '@/components/ui/button';

export function MyComponent() {
  return <Button variant="primary">点击我</Button>;
}
```

### 设计师

使用设计系统文档作为设计参考，确保设计稿符合系统规范。
