# 数据库种子数据系统

## 概述

数据库种子数据系统用于为不同场景生成测试和开发环境所需的数据。系统采用 **Fixture-first** 架构，结合真实基础数据和智能生成器，确保数据的真实性和可维护性。

### 为什么需要种子数据系统？

- **开发环境**：快速初始化本地数据库，无需手动创建测试数据
- **演示环境**：为 Vercel 部署提供丰富的演示数据
- **课程考试**：为课程评估提供精确可控的数据集

### 核心特性

- **真实数据**：61个真实城市、72个真实机场、39个真实航空公司
- **类型安全**：所有数据类型从数据库 Schema 自动推导，Schema 变更自动同步
- **场景驱动**：三种预设场景满足不同需求（开发、演示、考试）
- **灵活配置**：支持完全重置或增量添加数据
- **数据完整性**：自动处理外键关系和数据依赖

## 快速开始

### 基本使用

```bash
# 开发环境 - 适中数据量，快速启动
pnpm db:seed --scenario=development

# 演示环境 - 丰富数据，完整功能展示
pnpm db:seed --scenario=demo

# 考试环境 - 精确数据，用于课程评估
pnpm db:seed --scenario=exam
```

### 场景选择指南

| 我想... | 使用场景 | 命令 |

| 本地开发调试功能 | Development | `pnpm db:seed --scenario=development` |

```bash

# 场景2: 重置开发环境数据
pnpm db:seed --scenario=development  # 清空并重新生成

# 场景3: 准备演示环境

# 场景4: 在现有数据基础上添加更多航班
pnpm db:seed --scenario=development --mode=incremental
```

## 系统架构

### 设计理念

系统采用 **Fixture-first + Generator** 混合架构：

1. **Fixture-first**：基础数据（城市、机场、航空公司）使用真实数据
   - 数据质量高，符合真实业务场景
   - 便于调试和问题定位
   - 支持手动维护和更新

2. **Generator-based**：复杂数据（航班、订单）使用智能生成器
   - 自动生成大量测试数据
   - 保证数据关系正确性
   - 支持随机性和可重现性（通过 seed）

3. **Type-safe**：所有数据类型从 Schema 自动推导
   - Schema 变更自动反映到 Fixture 类型
   - 编译时类型检查，减少运行时错误
   - IDE 自动补全和类型提示

## 数据说明

### 真实基础数据（Fixtures）

系统内置了真实的航空业数据，确保测试环境接近生产环境：

#### 城市数据（61个）

包含国内外主要城市，每个城市包含：
- **IATA 代码**：3位标准代码（如 SHA、BJS、NYC）
- **城市名称**：中文或英文名称
- **时区**：标准时区（如 Asia/Shanghai）
- **国内/国际**：标记是否为国内城市
- **热门标记**：是否为热门旅游城市

**示例城市**：上海、北京、广州、深圳、成都、纽约、东京、伦敦、巴黎、悉尼...

#### 机场数据（72个）

真实的国际机场数据：
- **IATA 代码**：3位机场代码（如 PVG、PEK、NRT）
- **机场名称**：完整机场名称（如 浦东国际机场）
- **所属城市**：关联到城市数据

**示例机场**：浦东国际机场、首都国际机场、成田国际机场、希思罗机场...

#### 航空公司数据（39个）

主要航空公司数据：
- **IATA 代码**：2位航司代码（如 CA、MU、UA）
- **公司名称**：航空公司名称（如 中国国际航空）
- **Logo URL**：航司 Logo（如无则使用 UI Avatars 生成）

**示例航司**：中国国际航空、中国东方航空、美国联合航空、日本航空...

### 生成数据（Generators）

#### 航班数据（Flight Generator）

基于真实机场和航空公司生成航班：

**生成规则**：
- **航班号**：航司代码 + 4位数字（如 CA1234）
- **出发/到达机场**：从真实机场中随机选择（确保不同）
- **出发时间**：未来 7-90 天内随机时间
- **飞行时长**：根据距离估算（2-15小时）
- **机型**：使用 Faker 生成真实机型（如 Boeing 737-800）
- **航站楼**：随机生成（T1、T2、T3 等）

**座位舱位生成**：
每个航班生成 2-3 个舱位：

| 舱位 | 基础价格 | 座位数 | 可用座位 |
|------|----------|--------|----------|
| 经济舱 (ECONOMY) | ¥500-2000 | 150-200 | 总数的 60%-100% |
| 商务舱 (BUSINESS) | ¥2000-8000 | 20-40 | 总数的 50%-100% |
| 头等舱 (FIRST) | ¥8000-20000 | 8-16 | 总数的 40%-100% |

#### 用户和乘客数据（User Generator）

**用户生成**：
- **姓名**：使用 Faker 生成随机姓名
- **邮箱**：基于姓名生成（如 john.doe@example.com）
- **手机号**：70% 概率生成中国手机号
- **邮箱验证**：80% 概率已验证
- **头像**：50% 概率包含头像 URL

**乘客生成**：
每个用户生成 1-5 个乘客信息：
- **姓名**：中文或英文姓名
- **国籍**：支持 CN、US、JP、UK 等
- **性别**：male、female、other
- **出生日期**：1-80 岁随机年龄
- **证件类型**：
  - 身份证：18位数字
  - 护照：E + 8位数字
  - 其他：10位字母数字组合
- **证件有效期**：未来 1-10 年

#### 订单数据（Order Generator）

生成完整的订单流程数据：

**订单生成**：
- **订单号**：ORD + 时间戳 + 随机数
- **用户**：从已生成用户中随机选择
- **航班座位**：从可用座位中随机选择
- **乘客数量**：1-3 人
- **订单状态**：
  - PENDING_PAYMENT (20%)
  - CONFIRMED (60%)
  - CANCELLED (10%)
  - COMPLETED (10%)
- **附加服务**：随机选择餐食、保险、接送机等

**订单乘客**：
- 从用户的乘客列表中选择
- 快照乘客信息（姓名、证件等）

**支付记录**：
- **支付方式**：余额、支付宝、微信
- **支付状态**：SUCCESS、FAILED、PENDING
- **交易号**：TXN + 时间戳 + 随机数

#### 搜索历史数据（Search History Generator）

生成用户搜索行为数据：

**生成规则**：
- **用户**：每个用户 3-10 条搜索记录
- **出发/到达城市**：从真实城市中随机选择
- **出发日期**：未来 1-90 天
- **返程日期**：单程 (70%) 或往返 (30%)
- **乘客数量**：1-4 人
- **舱位偏好**：
  - 不限 (70%)
  - 经济舱 (20%)
  - 商务舱 (8%)
  - 头等舱 (2%)
- **搜索时间**：过去 30 天内随机时间

### 添加新的数据表支持

如果数据库 Schema 新增了表，需要添加种子数据支持：

#### 步骤 1：创建 Generator（如果需要）

在 `src/db/generators/` 目录下创建新的生成器文件：

```typescript
// src/db/generators/new-table-generator.ts
import { faker } from "@faker-js/faker";
import type { InferInsertModel } from "drizzle-orm";
import { newTable } from "@/db/schema/new-table";

type NewTableInsert = Omit<
  InferInsertModel<typeof newTable>,
  "id" | "createdAt" | "updatedAt"
>;

export interface NewTableGeneratorInput {
  // 定义输入参数
  count: number;
  seed?: number;
}

export function generateNewTableData(
  input: NewTableGeneratorInput
): NewTableInsert[] {
  if (input.seed) {
    faker.seed(input.seed);
  }

  const data: NewTableInsert[] = [];

  for (let i = 0; i < input.count; i++) {
    data.push({
      // 使用 Faker 生成数据
      field1: faker.lorem.word(),
      field2: faker.number.int({ min: 1, max: 100 }),
      // ...
    });
  }

  return data;
}
```

## 技术实现细节

### 类型安全系统

系统使用 Drizzle ORM 的类型推导功能，确保 Fixture 数据类型与数据库 Schema 保持同步：

```typescript
// src/db/fixtures/types.ts
import type { InferInsertModel, Table } from "drizzle-orm";

export type FixtureData<T extends Table> = Omit<
  InferInsertModel<T>,
  "id" | "createdAt" | "updatedAt"
>;
```

**优势**：
- Schema 变更自动反映到 Fixture 类型
- 编译时类型检查，减少运行时错误
- IDE 自动补全和类型提示
- 重构安全，字段重命名自动更新

**使用示例**：

```typescript
// src/db/fixtures/base/cities.ts
import type { InferInsertModel } from "drizzle-orm";
import { cities } from "@/db/schema/cities";

export type CityFixture = Omit<

>;

  {
```

### 数据关系处理

系统严格按照依赖关系顺序插入数据：

#### 插入顺序

```typescript
// 1. 基础数据（无依赖）
await db.insert(cities).values(cityData);
await db.insert(airlines).values(airlineData);

// 2. 依赖基础数据
await db.insert(airports).values(airportData); // 依赖 cities

// 3. 依赖多个表
await db.insert(flights).values(flightData); // 依赖 airports, airlines

// 4. 依赖航班
await db.insert(flightSeatClasses).values(seatClassData); // 依赖 flights

// 5. 用户相关
await db.insert(users).values(userData);
await db.insert(passengers).values(passengerData); // 依赖 users

// 6. 订单相关（依赖最多）
await db.insert(orders).values(orderData); // 依赖 users, flightSeatClasses
await db.insert(orderPassengers).values(orderPassengerData); // 依赖 orders
await db.insert(payments).values(paymentData); // 依赖 orders

// 7. 搜索历史
await db.insert(flightSearchHistory).values(searchData); // 依赖 users, cities
```
