# 颜色系统

本项目使用基于 Tailwind CSS 的颜色系统，采用 OKLCH 颜色空间以确保视觉一致性和更好的感知均匀性。

## 使用指南

### 在组件中使用颜色

使用 Tailwind 的语义化 class 名称：

```tsx
// 主要按钮
<button className="bg-primary text-primary-foreground hover:bg-primary/90">
  点击我
</button>

// 次要按钮
<button className="bg-secondary text-secondary-foreground hover:bg-secondary/80">
  取消
</button>

// 破坏性操作
<button className="bg-destructive text-destructive-foreground hover:bg-destructive/90">
  删除
</button>

// 卡片容器
<div className="bg-card text-card-foreground border border-border rounded-lg p-4">
  <h3 className="font-semibold">卡片标题</h3>
  <p className="text-muted-foreground">这是一段辅助说明文字</p>
</div>

// 输入框
<input className="bg-background text-foreground border border-input focus:ring-2 focus:ring-ring" />
```

### 透明度修饰符

可以使用 Tailwind 的透明度语法来调整颜色透明度：

```tsx
<div className="bg-primary/10">10% 透明度</div>
<div className="bg-primary/50">50% 透明度</div>
<div className="bg-primary/90">90% 透明度</div>
```

### 自定义颜色值

如需添加新的颜色，请在 `src/app/(frontend)/globals.css` 中修改 CSS 变量：

1. 在 `:root` 中添加浅色模式的颜色值
2. 在 `.dark` 中添加深色模式的颜色值
3. 在 `@theme inline` 中添加对应的 Tailwind 映射

示例：

```css
:root {
  /* 添加新颜色 */
  --success: oklch(0.6 0.15 150);
  --success-foreground: oklch(1 0 0);
}

.dark {
  --success: oklch(0.7 0.18 150);
  --success-foreground: oklch(0.1 0 0);
}

@theme inline {
  --color-success: var(--success);
  --color-success-foreground: var(--success-foreground);
}
```

## 最佳实践

### 1. 优先使用语义化颜色名称

✅ **推荐：**
```tsx
<button className="bg-primary text-primary-foreground">提交</button>
```

❌ **不推荐：**
```tsx
<button className="bg-[oklch(0.205_0_0)] text-white">提交</button>
```

### 2. 确保无障碍性

所有前景色和背景色的对比度应符合 WCAG AA 标准（至少 4.5:1）。本项目的颜色系统已经考虑了无障碍性要求。

### 3. 深色模式测试

所有颜色选择都应在浅色和深色模式下测试，确保在两种模式下都有良好的视觉效果。使用 Storybook 右上角的主题切换器进行测试。

### 4. 保持一致性

- 相同功能的元素使用相同的颜色
- 破坏性操作统一使用 `destructive` 颜色
- 辅助信息统一使用 `muted-foreground` 颜色

### 5. 适度使用颜色

- 避免在一个界面中使用过多颜色
- 保持视觉层次清晰
- 颜色应该引导用户注意力，而不是分散注意力

## OKLCH 颜色空间的优势

相比传统的 RGB 或 HSL 颜色空间，OKLCH 具有以下优势：

1. **感知均匀性**：亮度变化在视觉上更加均匀
2. **更宽的色域**：支持更多鲜艳的颜色
3. **更好的可预测性**：调整色相时亮度保持一致
4. **无障碍友好**：更容易创建符合对比度要求的颜色组合
